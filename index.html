Conway's Game of Life

Conway's game of life is a famous computer science problem first introduced by John Conway in 1970. Life is a “zero player” game that consists of an 'n' by 'm' checkerboard or grid, with each cell having the possibility of being on or off. The next state of each cell is decided by counting the number of neighboring cells that are on or off (see Rules). More information can be found on Wikipedia.

Algorithm

This code uses two arrays to store state for the current generation and the following generation. The nextGeneration() function iterates over each index on the array; it calls stayAlive() which checks the neighbors of each cell and returns value for the next state.

There are literally hundreds of algorithms for Conway's Game of Life ranging from the most simple (used in tihs project) to very complex (see Hashlife). If you're interested in learning about optimization of this algorithm I highly suggest started with Chapters 17 and 18 of Michael Abrash's classic Graphics Programming Black Book which is available for free.

Rules

These are the rules which determine if a cell dies or continues on to the next generation.

1.  Neighbors: The cells directly north, south, east and west of the current cell.

2.  Overcrowding: If a living cell has more than three living neighbors it will die during the transition to the next generation.

3.  Loneliness: If a living cell has less than two living neighboring cells it will die during the transition to the next generation.

4.  Resurrection: If a dead cell has more than three living neighbors it will come to live during the transition to the next generation.
